类型和函数

每个函数和表达式都有各自的类型，通常称一个表达式拥有类型T。

需要认识的是，类型是对计算机底层字节的抽象

Haskell的类型系统

三个方面

  • 强类型
  • 静态类型
  • 自动推导
    强类型
    强类型系统会导致系统拒绝任何无意义的表达式：比如将类型当作函数来使用，或者将字符串传给一个只接受参数的函数等。

另一个作用是，它不会自动地将值从一个类型转换到另一个类型，例如，将一个整数值传给一个接受浮点数地函数，C编译器回自动且静默地将参数从整数类型转换为浮点类型，而haskell编译器回引发一个编译错误。

要进行类型转换，必须显示地使用类型转换函数。

静态类型
静态类型系统指的是，编译器在编译期知道每个值和表达式的类型

一方面，强类型和静态类型保证了程序不可能发生运行时类型错误，一旦程序编译通过，程序的正确性就比其他语言来写要好得多（小错误会比较小）

另一方面，编译通过的haskell程序不会带有类型错误。

类型推导
haskell编译器可以自动推断程序中几乎所有表达式的类型。

一些常用的基本类型

  • Char
    单个Unicode字符
  • Bool
    表示一个布尔逻辑值，只有两个值：True和False
  • Int
    带符号的定长整数
  • Integer
    不限长度的带符号整数
  • Double
    用于表示浮点数
  • ::
    除了用来表示类型之外，还可以用于进行类型签名

函数

调用函数
先写出它的名字，后接函数的参数

函数的参数不需要用括号来包围，参数与参数之间不需要用逗号来隔开[使用空格就可以了]

函数应用的优先级比操作符要高，但是为了提高可读性，必须使用括号来让编译器知道。

复合数据类型

列表和元组诗最常用的复合数据类型

head 函数取出列表的第一个元素

tail 函数取出列表里除了第一个元素之外的其他元素

元组和列表非常不同，属性刚刚相反：列表可以任意长，且只能包含类型相同的值；元组的长度是固定的，但可以包含不同类型的值。

元组使用括号包围，元素之间用逗号分隔。

只有元组中的数量、位置和类型完全相同，两个元组才相等

元组通常用于两个地方：

  1. 函数需要返回多个值
  2. 使用定长容器，但没有必要使用自定义类型的时候

处理列表和元组的函数

take和drop
take返回一个包含list前n个元素的列表

drop则返回一个包含list丢弃了前n个元素之后，剩余元素的列表

fst则接受一个元组作为参数，返回元组的第一个元素

snd则接受一个元组作为参数，返回元组的第二个元素

将表达式传给参数

函数应用是左关联的

函数类型

使用:type命令可以查看函数的类型[缩写:t]

lines函数接受一个字符串作为输入，并将这个字符串按行转义字符分隔成多个字符串。

函数类型是右关联的：Haskell从右到左地串联起这些箭头，使用括号可以清晰地标示这个类型签名是怎样被解释的

Haskell源码以及自定义函数

Haskell 源码通常以.hs作为后缀

=号左边的add a b是函数名和函数参数，而右边的a+b则是函数体

将源码保存后，在ghci中使用:load(:l)载入

变量

Haskell中一旦给一个变量赋值，就不能再给这个变量赋其他值

条件求值

expr1和expr2必须是相同类型

null函数检查一个列表是否为空

Haskell里的多态

取出列表中的最后一个元素，则可以使用last函数

函数的类型签名里包含类型变量，那么久表示这个函数的某些参数可以是任意类型，我们称这些函数是多态的。
